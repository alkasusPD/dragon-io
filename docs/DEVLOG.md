# 개발 기록

새 작업은 날짜별로 맨 위에 추가한다. 무엇을 바꿨는지뿐 아니라 왜 바꿨는지도 기록한다.

## 2026-07-06

- 손가락이 플레이어 드래곤과 전방 시야를 가리지 않도록 모바일 이동 입력을 화면 좌측 하단 영역으로 분리.
- 플레이어의 최대 하강 위치를 화면 높이의 72%로 제한해 조이스틱을 잡은 손가락과 항상 간격을 유지.
- 버추얼 패드의 비례 감속을 제거하고 8px 데드존 이후 즉시 최고 속도로 이동하도록 변경해 키보드와 반응성을 통일.
- 조이스틱 반경을 72px에서 56px로 줄여 방향 전환에 필요한 손가락 이동 거리를 단축.
- 안드로이드 화면을 몰입형 전체 화면으로 전환하고 기기 화면비에 맞춰 캔버스 높이를 확장해 상단 여백 제거.
- 버추얼 패드 생성 영역 제한을 제거해 게임 화면 어디를 터치해도 패드가 나타나도록 수정.
- 터치 이동 속도를 40% 높이고 최신 포인터 좌표를 우선 반영해 조작 지연 체감 완화.
- 모바일 궁극기 버튼을 하단 204px에서 280px로 올려 이동 조작 영역과 분리.
- 안드로이드 카메라 컷아웃 영역까지 배경을 렌더링하고 HUD만 안전 영역 아래에 유지.
- 모바일 게임 캔버스를 원래 9:16 비율로 복원해 캐릭터와 투사체의 크기 왜곡 제거.
- 세로로 긴 화면의 추가 상단 영역은 별도 배경 이미지만 채우고 HUD는 전체 화면 상단에 배치.
- 모바일 궁극기 위치 규칙의 우선순위를 높여 기본 하단 204px 값이 덮어쓰는 문제 수정.
- 상단 확장 배경의 위치를 캔버스 배경 스크롤과 동기화해 9:16 경계에서 이미지가 끊기는 현상 제거.
- 기존 9:16 배경을 억지로 이어 붙이는 방식을 폐기하고, 원본 스타일을 유지한 841×1870(약 20:9) 안드로이드 전용 연속 배경을 새로 제작.
- 모바일의 상단 확장 영역과 게임 캔버스가 동일한 긴 배경 자산을 공유하도록 변경.
- 안드로이드 배경을 3:1 비율의 724×2172 버전으로 다시 제작해 기존 전용 배경보다 세로 범위를 약 35% 확장.
- 3:1 배경의 상단과 하단 지형 폭·구름·수로가 연결되도록 세로 타일 전용 버전으로 다시 제작.
- 모바일 게임 캔버스를 실제 전체 화면 비율로 확장하되 가로 논리 해상도 540을 유지해 스프라이트 크기 보존.
- 몬스터 생성 기준을 전체 화면 최상단으로 이동시켜 상단 배경 아래 경계에서 갑자기 나타나는 문제 제거.
- 몬스터의 플레이어 투사체용 판정을 가로 1.55배·세로 1.12배 타원으로 확장해 날개에 맞아 보이는 탄환이 빗나가는 문제 완화.
- 플레이어와 몬스터의 몸통 충돌 판정은 기존 원형 반경을 유지해 회피 난이도 변화 방지.
- 강화 선택 첫 슬롯이 보유 스킬로 고정되던 규칙을 변경해, 보유 스킬과 신규 스킬이 모두 있을 때 각각 50% 확률로 등장.
- 첫 슬롯에서 신규 스킬이 선택된 경우 두 번째 슬롯도 다른 신규 스킬을 우선 제시해 새 빌드 탐색 기회 확대.
- 챕터 1 마지막을 웨이브 10으로 정의하고 에메랄드 드레드윙 페이즈 2를 최종 보스로 추가.
- 최종 보스용 12프레임 256px 가로 스프라이트와 보라색 신체 오라 적용.
- 최종 보스 패턴 1은 빠른 보라색 에너지 구체 3~5발을 부채꼴로 발사.
- 최종 보스 패턴 2는 느리게 플레이어를 추적하는 대형 호밍 에너지 구체 2기를 발사.
- 호밍 구체에 이중 발광, 회전 고리, 외곽 스파이크, 보라색 잔상을 적용해 보스 탄환의 위압감 강화.
- 최종 보스 처치 시 챕터 클리어 결과 화면으로 전환.
- 좌측 하단을 누른 위치에 플로팅 조이스틱이 나타나는 상대 이동 방식을 유지.
- 조작 복잡도를 낮추기 위해 하강 버튼과 하강 회피 시스템 제거.
